
Brokerage Deal Intake Assistant 
1. Product Goal

Build a private internal web app that takes a real estate transaction package PDF and automatically creates a draft **Deal Information Sheet** for admin review.

The workflow is:

Upload transaction package PDF
→ classify pages/documents
→ extract deal fields
→ cross-check values
→ admin reviews/edits
→ export Deal Information Sheet / CSV 

---

2. Core Use Case

The brokerage receives a 20–50 page PDF package containing documents such as:

* Deal Information Sheet
* Agreement of Purchase and Sale
* OREA Form 801
* OREA Form 320
* Deposit proof
* FINTRAC / ID forms
* Lease agreement
* Disclosure forms

The sample PDFs include this kind of multi-document transaction package.   

The app should help admins generate and complete the Deal Information Sheet faster.

Show extracted fields in editable form.

Each field should have:

```ts

  value: string | number | null;
  confidence: "high" | "medium" | "low";
```

UI behavior:

```text

Yellow = needs review
Red = missing
```

# 8. Security Requirements

These PDFs contain sensitive transaction, identity, signature, and financial information.

Minimum security:

```text
Supabase Auth required
Row Level Security enabled
Private storage buckets only
Signed URLs only
Audit logs
No public file URLs
Auto-delete original PDFs after configurable period
```

Recommended retention:

```text
Original PDFs: delete after 7–30 days
Extracted/reviewed deal record: retain unless brokerage requests deletion
Audit logs: retain
```

Important product promise:

```text
Uploaded documents are used only to process the brokerage’s transaction package and are not used to train models.
```

# 11. Acceptance Criteria for MVP

```text
Admin uploads one transaction package.
System extracts major deal fields.
Admin can correct fields.
```

